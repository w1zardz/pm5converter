What Changed in PM5?

Level → World

The entire pocketmine\level namespace was renamed to pocketmine\world. All Level references, including Server methods like getLevels(), are now getWorldManager()->getWorlds().

Numeric IDs Removed

No more Item::get(256) or Block::get(1). Use VanillaItems::IRON_SHOVEL() and VanillaBlocks::STONE() instead. String parsing via StringToItemParser.

Event API Overhaul

setCancelled(true) becomes cancel(). Events are called via $event->call() instead of PluginManager->callEvent(). Handlers skip cancelled events by default.

Entity System Rewrite

No more Entity::createEntity(). Entities use direct constructors with Location. NBT is handled via initEntity()/saveNBT() instead of public namedtag.

Player API Split

Player moved to pocketmine\player\Player. Network methods moved to getNetworkSession(). Effects, hunger, XP use sub-managers.

Task Signatures

Task->onRun($currentTick) becomes onRun(): void. PluginTask removed — use Task or ClosureTask. AsyncTask callbacks lost their Server parameter.

Conversion Rules

Namespace Replacements

  • pocketmine\level\* → pocketmine\world\*
  • pocketmine\tile\* → pocketmine\block\tile\*
  • pocketmine\Player → pocketmine\player\Player
  • pocketmine\utils\UUID → Ramsey\Uuid\Uuid

Method Conversions

  • onActivate() → onInteract()
  • getLevel() → getWorld()
  • addEffect() → getEffects()->add()
  • getFood() → getHungerManager()->getFood()

Visibility & Signatures

  • public function onEnable() → protected function onEnable(): void
  • onRun($currentTick) → onRun(): void
  • onCompletion(Server $s) → onCompletion(): void

Frequently Asked Questions

What is PM5 Converter?
PM5 Converter is a free, open-source, browser-based tool that automatically converts PocketMine-MP plugin PHP code from API 3.x/4.x to API 5.x. It applies 120+ conversion rules covering namespace changes, method renames, event API updates, and more. No server upload required — everything runs in your browser.
Will it convert my plugin perfectly?
The converter handles most common API changes automatically, including namespace migrations, method renames, event API changes, and plugin.yml updates. However, complex patterns like custom entity registration, advanced NBT manipulation, or Item/Block numeric ID lookups may need manual adjustments. Always review the conversion log and test thoroughly.
What PocketMine-MP versions are supported?
The converter supports input code written for PocketMine-MP API 3.x (PM3) and API 4.x (PM4), converting it to API 5.x (PM5). It covers the full migration path including PM3→PM4 and PM4→PM5 changes in a single pass.
Is my code safe? Does it upload anywhere?
Your code never leaves your device. PM5 Converter runs entirely in your browser using JavaScript. No data is sent to any server. The tool is open-source — you can inspect the code on GitHub.
Can I convert multiple files at once?
Yes! You can drag and drop multiple .php files onto the upload area. Each file will be converted separately, and you can download all results. For large plugins, consider converting file by file to review changes carefully.
Does it handle plugin.yml changes?
Yes. When you enable the "plugin.yml" option, the converter updates the API version to 5.0.0, adjusts permission defaults, and flags deprecated features like nested permissions.
